# 静态代理

```java
public interface UserService {
    public void select();   
}

public class UserServiceImpl implements UserService {  
    public void select() {  
        System.out.println("查询 selectById");
    }
}
```

代理类

```java
public class UserServiceProxy implements UserService {
    public UserServiceProxy(UserService target) {
        this.target = target;
    }
    public void select() {
        before();
        target.select();    // 这里才实际调用真实主题角色的方法
        after();
    }

    private void before() {     // 在执行方法之前执行
        System.out.println(String.format("log start time [%s] ", new Date()));
    }
    private void after() {      // 在执行方法之后执行
        System.out.println(String.format("log end time [%s] ", new Date()));
    }
}
```
客户端测试

```java
  public class Client {
      public static void main(String[] args) {
          UserService userServiceImpl = new UserServiceImpl();
          UserService proxy = new UserServiceProxy(userServiceImpl); proxy.select();
     			proxy.select();
 			 }
 }
```
优点：简单、没有侵入源代码达到了增强的目的。

缺点：代理多个类的时候，代理对象与目标对象一致的接口

​	1、维护一个代理类，代理类实现多个接口-代理类过于庞大

​	2、新增代理类，每个目标对象一个代理类-代理类过多

​	3、接口新增方法时，目标对象的代理类要同时修改，不易维护



# 动态代理

Java虚拟机类加载过程主要分为五个阶段：加载、验证、准备、解析、初始化。



```java
import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.util.Date;

public class LogHandler implements InvocationHandler {
    Object target;  // 被代理的对象，实际的方法执行者
 		public LogHandler(Object target) {
        this.target = target;
    }
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        before();
        Object result = method.invoke(target, args);  // 调用 target 的 method 方法
        after();
        return result;  // 返回方法的执行结果
    }
    // 调用invoke方法之前执行
    private void before() {
        System.out.println(String.format("log start time [%s] ", new Date()));
    }
    // 调用invoke方法之后执行
    private void after() {
        System.out.println(String.format("log end time [%s] ", new Date()));
    }
}
```
